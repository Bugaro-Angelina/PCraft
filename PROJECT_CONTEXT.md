# PCraft Project Context

## Краткое описание

PCraft — мобильное Android-приложение для подбора компьютерных комплектующих и сборки ПК.  
Пользователь может:

- просматривать каталог комплектующих;
- открывать карточку товара;
- смотреть характеристики, описание, цены и магазины;
- добавлять товары в избранное;
- добавлять комплектующие в конструктор ПК;
- проверять совместимость сборки;
- выбирать магазин для каждого комплектующего;
- сохранять сборки;
- входить в аккаунт по email и паролю.

## Технологический стек

- Kotlin
- Android Studio
- Jetpack Compose
- Navigation Compose
- MVVM
- Hilt
- Room
- Coil
- Firebase модуль подключён архитектурно, но текущая рабочая авторизация и пользовательские данные сохраняются локально

## Архитектура

Проект разделён на слои:

- `data`
  - mock-данные
  - модели
  - DAO
  - репозитории
- `domain`
  - use case для совместимости, цен и нагрузки БП
- `ui`
  - экраны
  - компоненты интерфейса
  - view model
- `di`
  - dependency injection через Hilt

## Основные экраны

1. Главная
   - каталог комплектующих
   - поиск
   - фильтрация по типу
   - сортировка

2. Карточка комплектующего
   - изображение
   - описание
   - характеристики
   - цены магазинов
   - выбор магазина
   - добавление в избранное
   - добавление в конструктор

3. Конструктор
   - слоты комплектующих
   - быстрая замена
   - удаление компонента
   - минимальная цена сборки
   - цена по выбранным магазинам
   - нагрузка блока питания
   - фильтр по совместимости
   - название и заметки
   - переход к экрану совместимости
   - переход к экрану со ссылками и ценами

4. Экран отчёта совместимости
   - плюсы сборки
   - предупреждения
   - проблемы
   - статус по каждому комплектующему

5. Избранное
   - список добавленных пользователем товаров

6. Аккаунт
   - вход и регистрация
   - история сохранённых сборок
   - удаление сохранённых сборок

## Логика совместимости

Сейчас реализована проверка:

- совместимости процессора и материнской платы по сокету;
- совместимости оперативной памяти и материнской платы по типу памяти;
- совместимости корпуса и материнской платы по форм-фактору;
- достаточности мощности блока питания;
- отображения общей сводки проблем;
- статуса каждой детали:
  - `Совместимо`
  - `Частично совместимо`
  - `Не совместимо`

## Цены и магазины

Для каждого комплектующего в mock data задано несколько предложений магазинов.

Поддерживается:

- минимальная цена комплектующего;
- выбор одного магазина для конкретного товара;
- расчёт минимальной цены всей сборки;
- расчёт цены по выбранным магазинам;
- отдельный экран со списком ссылок и цен всей сборки.

## Избранное

Избранное работает отдельно для каждого состояния пользователя:

- у гостя свой список;
- у зарегистрированного пользователя свой список.

Удаление из избранного сохраняется и не должно возвращаться после обновления состояния.

## Авторизация

Сейчас реализована простая авторизация по email и паролю без подтверждения почты:

- регистрация создаёт нового пользователя;
- вход проверяет наличие пользователя и совпадение пароля;
- сессия сохраняется локально;
- у нового пользователя не должно быть предзаполненных сохранённых сборок и избранного.

## Хранение данных

Сейчас проект использует:

- локальные mock data для каталога;
- локальное сохранение аккаунтов;
- локальное сохранение избранного;
- локальное сохранение сборок по пользователю;
- локальные изображения товаров в ресурсах приложения.

Часть архитектуры подготовлена под Firebase/Firestore, но основной рабочий сценарий сейчас не зависит от обязательной настройки облака.

## Визуальная концепция

- тёмный интерфейс;
