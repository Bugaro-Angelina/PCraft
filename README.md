# PCraft - Android PC Configurator App

## Описание проекта

PCraft - это мобильное приложение для подбора и конфигурации компьютерных комплектующих. Приложение позволяет пользователям:

- Просматривать каталог комплектующих (CPU, GPU, RAM, PSU и т.д.)
- Добавлять комплектующие в избранное
- Собирать конфигурацию ПК в конструкторе
- Проверять совместимость компонентов
- Просматривать предложения от различных магазинов
- Рассчитывать общую стоимость сборки
- Сохранять конфигурации в истории

## Технологический стек

- **Язык**: Kotlin
- **UI**: Jetpack Compose
- **Архитектура**: MVVM (Model-View-ViewModel)
- **Паттерны**: Repository Pattern, Use Case Pattern
- **Хранилище**: Room Database
- **Инъекция зависимостей**: Hilt
- **Навигация**: Navigation Compose
- **Управление состоянием**: StateFlow
- **Дизайн**: Material 3 с кастомной темой (dark mode, neon accents)

## Архитектура

### Data Layer

- **Models**: Основные классы данных (Component, StoreOffer, BuildConfiguration)
- **DAO (Data Access Objects)**: Интерфейсы для работы с базой данных
- **Database**: Room DatabaseConfiguration
- **Repository**: Абстракция над источниками данных (DB, Mock Data)
- **MockDataProvider**: Тестовые данные с реальными компонентами

### Domain Layer

- **UseCase**: Business logic, отделённая от UI
  - `CheckCompatibilityUseCase`: Проверка совместимости компонентов
  - `CalculateBuildPriceUseCase`: Расчёт стоимости сборки
  - `CalculatePsuLoadUseCase`: Расчёт нагрузки на БП

### UI Layer

- **Screens**: 
  - HomeScreen: Каталог компонентов с поиском и фильтрами
  - DetailsScreen: Детали компонента и предложения магазинов
  - BuilderScreen: Конструктор ПК с выбором компонентов
  - CompatibilityReportScreen: Подробный отчёт о совместимости
  - FavoritesScreen: Избранные компоненты
  - ProfileScreen: История сборок
  
- **Components**:
  - ComponentCard: Карточка компонента в каталоге
  - CompatibilityStatusIndicator: Индикатор статуса совместимости
  - PsuLoadIndicator: Визуализация нагрузки на БП с прогресс-баром

- **Theme**: Кастомная Material 3 тема с тёмным фоном и неоновыми акцентами

### ViewModels

- StateFlow для управления состоянием
- Использование Hilt для инъекции зависимостей
- Методы для обновления состояния и вызова use cases

## Функциональность

### 1. Каталог компонентов
- Поиск по названию
- Фильтрация по типу (CPU, GPU, RAM и т.д.)
- Добавление в избранное

### 2. Детали компонента
- Полная информация и характеристики
- Предложения от нескольких магазинов (DNS, Ситилинк, Регард)
- Выбор магазина через checkbox
- Кнопка добавления в конструктор

### 3. Конструктор ПК
- Слоты для каждого типа компонента
- Отображение выбранных компонентов и цен
- Расчёт общей стоимости (минимальная и по выбранным магазинам)
- Визуализация нагрузки на БП с цветовой кодировкой:
  - Зелёный: < 70%
  - Жёлтый: 70-85%
  - Красный: > 85%
- Фильтр по совместимости (Все / Совместимые / Проблемные)
- Проверка совместимости с подробным отчётом

### 4. Отчёт совместимости
- Общая оценка сборки
- Нагрузка на БП с процентом загрузки
- Блоки:
  - ✓ Плюсы сборки
  - ⚠ Предупреждения
  - ❌ Проблемы
- Статус каждого компонента с объяснениями

### 5. Проверка совместимости

Реализована полная логика проверки:

1. **Сокет CPU и материнская плата**: Должны совпадать
2. **Тип RAM и материнская плата**: RAM должна поддерживаться платой
3. **Форм-фактор материнской платы и корпус**: Плата должна поместиться в корпус
4. **Нагрузка на БП**: 
   - INCOMPATIBLE: потребление > мощность БП
   - PARTIALLY_COMPATIBLE: нагрузка > 85%
   - COMPATIBLE: всё в порядке

5. **Обработка неполных данных**:
   - Если данных не хватает, статус = PARTIALLY_COMPATIBLE
   - Показывается сообщение: "Недостаточно данных для полной проверки"

### 6. Расчёт цены

Две стоимости:
- **Минимальная**: сумма минимальных цен всех компонентов
- **По выбранным магазинам**: использует выбранные цены, для невыбранных берёт минимум
