# PCraft - Приложение для конфигурации ПК

## О проекте

PCraft - мобильное приложение для подбора и конфигурации компьютерных комплектующих. С его помощью можно:

- Просматривать огромный каталог компонентов (процессоры, видеокарты, память, блоки питания и многое другое)
- Добавлять понравившиеся вещи в избранное
- Собирать свою идеальную конфигурацию ПК в специальном конструкторе
- Проверять, совместимы ли компоненты друг с другом
- Смотреть предложения от разных магазинов
- Рассчитывать, сколько всё это будет стоить
- Сохранять свои сборки в истории, чтобы не потерять

## Какие технологии использованы

- **Язык программирования**: Kotlin
- **Интерфейс**: Jetpack Compose
- **Архитектура**: MVVM (Model-View-ViewModel)
- **Паттерны**: Repository Pattern и Use Case Pattern
- **Локальное хранилище**: Room Database и SharedPreferences
- **Облачное хранилище**: Firebase Firestore
- **Инъекция зависимостей**: Hilt
- **Навигация**: Navigation Compose
- **Управление состоянием**: StateFlow
- **Дизайн**: Material 3 с кастомной темой (тёмный режим и неоновые акценты)

## Архитектура приложения

### Слой данных (Data Layer)

- **Модели**: Основные классы данных, такие как Component, StoreOffer, BuildConfiguration
- **DAO (Data Access Objects)**: Интерфейсы для взаимодействия с локальной базой данных
- **Room Database**: Локальное хранилище компонентов и сборок
- **SharedPreferences**: Хранилище избранных компонентов и сессий пользователя
- **Firebase Firestore**: Облачное хранилище каталога компонентов, предложений магазинов и синхронизация избранного между устройствами
- **Repository**: Абстракция над всеми источниками данных с автоматической синхронизацией
- **MockDataProvider**: Тестовые данные для разработки и инициализации

### Доменный слой (Domain Layer)

- **UseCase**: Бизнес-логика, отделённая от интерфейса
  - `CheckCompatibilityUseCase`: Проверяет совместимость компонентов
  - `CalculateBuildPriceUseCase`: Считает стоимость сборки
  - `CalculatePsuLoadUseCase`: Рассчитывает нагрузку на блок питания

### Слой интерфейса (UI Layer)

- **Экраны**:
  - HomeScreen: Каталог компонентов с поиском и фильтрами
  - DetailsScreen: Подробности о компоненте и предложения магазинов
  - BuilderScreen: Конструктор ПК для выбора компонентов
  - CompatibilityReportScreen: Детальный отчёт о совместимости
  - FavoritesScreen: Избранные компоненты
  - ProfileScreen: История сборок

- **Компоненты**:
  - ComponentCard: Карточка компонента в каталоге
  - CompatibilityStatusIndicator: Индикатор статуса совместимости
  - PsuLoadIndicator: Визуализация нагрузки на БП с прогресс-баром

- **Тема**: Кастомная тема Material 3 с тёмным фоном и неоновыми акцентами

### ViewModels

- Использование StateFlow для управления состоянием
- Hilt для инъекции зависимостей
- Методы для обновления состояния и вызова use cases

## Что умеет приложение

### 1. Каталог компонентов
- Поиск по названию
- Фильтрация по типу (CPU, GPU, RAM и т.д.)
- Добавление в избранное одним тапом

### 2. Детали компонента
- Вся информация и характеристики
- Предложения от разных магазинов (DNS, Ситилинк, Регард)
- Выбор магазина через чекбоксы
- Кнопка для добавления в конструктор

### 3. Конструктор ПК
- Слоты для каждого типа компонента
- Показ выбранных компонентов и их цен
- Расчёт общей стоимости (минимальная и по выбранным магазинам)
- Визуализация нагрузки на БП с цветами:
  - Зелёный: меньше 70%
  - Жёлтый: 70-85%
  - Красный: больше 85%
- Фильтр по совместимости (Все / Совместимые / Проблемные)
- Проверка совместимости с подробным отчётом

### 4. Отчёт совместимости
- Общая оценка сборки
- Нагрузка на БП с процентом
- Блоки:
  - ✓ Плюсы сборки
  - ⚠ Предупреждения
  - ❌ Проблемы
- Статус каждого компонента с объяснениями

### 5. Проверка совместимости

Реализована полная логика проверки:

1. **Сокет CPU и материнская плата**: Должны совпадать
2. **Тип RAM и материнская плата**: Память должна поддерживаться платой
3. **Форм-фактор материнской платы и корпус**: Плата должна влезть в корпус
4. **Нагрузка на БП**:
   - INCOMPATIBLE: потребление больше мощности БП
   - PARTIALLY_COMPATIBLE: нагрузка больше 85%
   - COMPATIBLE: всё нормально

5. **Обработка неполных данных**:
   - Если данных не хватает, статус = PARTIALLY_COMPATIBLE
   - Показывается сообщение: "Недостаточно данных для полной проверки"

### 6. Расчёт цены

Две стоимости:
- **Минимальная**: Сумма минимальных цен всех компонентов
- **По выбранным магазинам**: Использует выбранные цены, для остальных берёт минимум
